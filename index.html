Rules & Guidelines

1. Event Duration

The Vibe Coding contest is a 48-hour event. The timer starts for you individually as soon as you click the "Deploy Countdown" button. This action is final and cannot be reversed.

2. Submission

All project files must be submitted via the "Submission Portal" *before* your personal 48-hour timer expires. Late submissions will be automatically rejected by the system.

You must submit a link to a public GitHub repository containing your project and a link to the live, deployed website.

3. Theme

The theme of the contest is **"Transform"**. Your project must interpret this theme. This could relate to data transformation, UI transformation, a personal transformation story, or the literal Transformers franchise. Be creative!

4. Originality & AI

Your project must be your own original work, created within the 48-hour window. You are permitted to use AI tools (like Gemini, GitHub Copilot, etc.) to *assist* you, but your core concept, logic, and implementation must be your own. You must disclose the use of AI tools in your project's `README.md` file.

5. Code of Conduct

All participants must adhere to the Mozilla Community Participation Guidelines. Be respectful, supportive, and collaborative. Any form of harassment will result in immediate disqualification.

Permitted Tools & Tech

You have full creative freedom. You can use any frameworks, libraries, and technologies you are comfortable with.

Suggestions

  • Frontend: React, Vue, Svelte, Angular, or even plain HTML/CSS/JS.
  • Backend: Node.js, Python (Django, Flask), Firebase, Supabase, etc.
  • Styling: Tailwind CSS, Bootstrap, Chakra UI, or your own custom CSS.
  • AI Tools: You are welcome to use AI assistants like Gemini, Copilot, etc., to help you code, but you must disclose their use in your `README.md`.
